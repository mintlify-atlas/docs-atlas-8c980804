Selenide is a Java framework for writing UI tests that are easy to read and easy to maintain. It wraps Selenium WebDriver with a concise fluent API, built-in automatic waiting for dynamic content, and natural-language assertions — so you focus on test logic, not WebDriver boilerplate.

```java
@Test
public void login() {
  open("/login");
  $(By.name("user.name")).setValue("johny");
  $("#submit").click();
  $("#username").shouldHave(text("Hello, Johny!"));
}
```

## Why Selenide?

<CardGroup cols={2}>
  <Card title="Automatic waiting" icon="clock">
    No more `Thread.sleep()`. Selenide waits for elements automatically using a configurable timeout and polling interval.
  </Card>
  <Card title="Readable assertions" icon="check">
    `shouldBe(visible)`, `shouldHave(text("Hello"))` — assertions read like plain English and produce clear failure messages.
  </Card>
  <Card title="Ajax-friendly" icon="bolt">
    Built for real-world apps. Selenide retries failed conditions until the timeout expires, handling dynamic content gracefully.
  </Card>
  <Card title="Built-in screenshots" icon="camera">
    On every test failure, Selenide automatically captures a screenshot and saves the page source for diagnosis.
  </Card>
</CardGroup>
