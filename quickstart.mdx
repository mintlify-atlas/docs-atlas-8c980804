---
title: "Quick Start"
description: "Write your first Selenide test in minutes"
---

This guide walks you from zero to a working browser test. You need Java 11+ and Maven or Gradle.

<Steps>
  <Step title="Add the dependency">
    Add Selenide to your project. It brings Selenium WebDriver as a transitive dependency — you do not need to add it separately.

    <CodeGroup>
    ```xml Maven pom.xml
    <dependency>
        <groupId>com.codeborne</groupId>
        <artifactId>selenide</artifactId>
        <version>7.15.1</version>
        <scope>test</scope>
    </dependency>
    ```

    ```groovy Gradle build.gradle
    testImplementation 'com.codeborne:selenide:7.15.1'
    ```

    ```kotlin Gradle Kotlin build.gradle.kts
    testImplementation("com.codeborne:selenide:7.15.1")
    ```
    </CodeGroup>

    <Note>
      Selenide manages WebDriver binaries automatically via Selenium Manager (bundled with Selenium 4). You do not need to download ChromeDriver or GeckoDriver manually.
    </Note>
  </Step>

  <Step title="Add the static imports">
    Add these two static imports to your test class. They bring the entire Selenide API into scope.

    ```java
    import static com.codeborne.selenide.Selenide.*;
    import static com.codeborne.selenide.Condition.*;
    ```

    With these imports, you can call `open()`, `$()`, `$$()` and use conditions like `visible`, `text("...")` directly — no class prefixes needed.
  </Step>

  <Step title="Configure the base URL">
    Set `Configuration.baseUrl` to your application's base URL. All calls to `open("/path")` resolve relative to this URL.

    ```java
    Configuration.baseUrl = "https://the-internet.herokuapp.com";
    ```

    You can set this once in a `@BeforeAll` method, or globally in a base test class.
  </Step>

  <Step title="Write a test">
    Here is a complete JUnit 5 test that opens a login page, fills in credentials, and asserts the result:

    ```java LoginTest.java
    import com.codeborne.selenide.Configuration;
    import org.junit.jupiter.api.BeforeAll;
    import org.junit.jupiter.api.Test;

    import static com.codeborne.selenide.Selenide.*;
    import static com.codeborne.selenide.Condition.*;

    public class LoginTest {

        @BeforeAll
        static void setup() {
            Configuration.baseUrl = "https://the-internet.herokuapp.com";
        }

        @Test
        void successfulLogin() {
            open("/login");
            $("#username").setValue("tomsmith");
            $("#password").setValue("SuperSecretPassword!");
            $("[type=submit]").click();
            $(".flash.success").shouldHave(text("You logged into a secure area!"));
        }

        @Test
        void failedLogin() {
            open("/login");
            $("#username").setValue("wrong");
            $("#password").setValue("wrong");
            $("[type=submit]").click();
            $(".flash.error").shouldBe(visible);
        }
    }
    ```

    Notice:
    - `open("/login")` — opens the page (relative to `baseUrl`)
    - `$("css")` — finds a single element
    - `.setValue(text)` — types into an input
    - `.click()` — clicks the element
    - `.shouldHave(text(...))` — asserts the element contains a text substring
    - `.shouldBe(visible)` — asserts the element is visible
    - All assertions wait automatically (default: 4 seconds) before failing
  </Step>

  <Step title="Run the test">
    <CodeGroup>
    ```bash Maven
    mvn test -Dtest=LoginTest
    ```

    ```bash Gradle
    ./gradlew test --tests LoginTest
    ```
    </CodeGroup>

    Selenide opens Chrome (the default browser), runs through your test steps, and closes the browser when done. If an assertion fails, a screenshot and the page HTML are saved automatically to `build/reports/tests/`.

    <Tip>
      Run headless in CI: set `Configuration.headless = true` in your setup method, or pass `-Dselenide.headless=true` on the command line.
    </Tip>
  </Step>
</Steps>
